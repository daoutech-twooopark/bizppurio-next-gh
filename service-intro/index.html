연동 방식

발송은 두 가지 방식 중 선택하며, 자원(템플릿·발신프로필 등)은 관리 API로 자동화할 수 있습니다.

BIZAPI
메시지 API
HTTPS REST 단일 엔드포인트(POST /v3/message)로 모든 채널 발송. 클라우드 호스팅.
BIZCLIENT
BIZCLIENT
biz_msg 큐 테이블에 INSERT하면 온프레미스 모듈이 폴링해 발송. 사내망·대량 배치에 적합.
KAPI / RAPI / NAPI
자원 관리 API
템플릿·발신프로필·챗봇·이미지 등 발송 자원을 코드로 등록·관리.

인증 흐름

메시지 API는 Basic 인증으로 토큰을 발급받아 Bearer로 발송합니다. 토큰은 24시간 유효하므로 캐싱해 재사용합니다.

1인증 요청
POST /v1/token
Authorization: Basic
base64(account:password)
→
2토큰 수신
{ "accesstoken": "...",
  "expired": "24h" }
→
3메시지 발송
POST /v3/message
Authorization:
Bearer {accesstoken}

서비스별 인증 방식

서비스인증 방식
메시지 APIBasic 인증 → Bearer 토큰 (24시간 유효)
카카오 KAPI요청 본문에 bizId + apiKey
네이버 NAPIrefreshToken → accessToken (Bearer)
BIZCLIENTDB 접근 권한 (별도 토큰 불필요)
